xBrain is an open-source AI idea engine built in Python that uses Claude to generate, score, stress-test, and refine project ideas through a multi-phase pipeline. Here is what it currently does:

PIPELINE PHASES:
1. META-LEARN (every 3 runs): Reads past run metrics, scores, kill reasons, and attack patterns. Distills a compact playbook with score calibration multipliers that correct scoring bias in later phases.
2. CONSTRAINT CHECK: When 2+ constraints are provided, detects logical contradictions and warns (non-blocking).
3. DIVERGE (Round 1): Generates N raw idea seeds using 6 techniques simultaneously — Domain Scan, Cross-Domain Collision, Contrarian Inversion, Contextual Constraints, AI-Augmentable Gap Detection, and Mechanism Stealing. Uses temp=0.9.
4. DEDUP: Collapses near-identical ideas, identifies over-represented themes and gap areas.
5. GAP-FILL (Round 2): Generates new ideas only in gap areas at temp=0.95, avoiding over-represented themes.
6. CONVERGE: Clusters, scores (8 dimensions: impact, confidence, sustainability, defensibility, market_timing, effort, cost, ethical_risk), ranks, and selects top N. Applies post-LLM calibration from meta-learning. Includes inverse fragility check per idea.
7. STRESS TEST: Single-round adversarial attack — each idea tested in parallel. Web search (DuckDuckGo + HackerNews) finds real prior art before attacking. 9 structured attack angles + 1 freeform devastating attack. Produces feasibility matrix, kill criteria, and verdict (BUILD/MUTATE/KILL/INCUBATE).
8. REFINE (if no BUILD verdicts): Up to 3 rounds of re-diverge with blocklists, mutations, and learnings from previous attacks. Progressively lower creativity.
9. MEMORY UPDATE: Persists idea archive, kill log, mutation archive, attack patterns, failure taxonomy, domain heat map, idea lineage, idea genes, and run metrics to xbrain-memory/persistent/.

SECOND PIPELINE — SPECIFY: Converts a BUILD idea into an actionable project spec with user stories, API contracts, data models, task breakdowns, risks, MVP scope, and kill criteria.

ADDITIONAL FEATURES:
- Export to CSV/Markdown/Jira-JSON for project management tools
- Idea lineage tracking across runs
- Idea genes: high-scoring ideas decomposed into reusable problem/solution patterns
- --strategy flag for model routing (single/cheapest/balanced/best) to control costs
- --dry-run for cost preview without API calls
- Brief auto-detection (file path or inline text)
- Per-phase cost breakdown in every report
- Automatic retry with exponential backoff for API rate limits

CURRENT COST PROFILE:
- ~12 API calls per run
- $0.10-$0.15 with Haiku, $0.20-$0.40 with Sonnet
- Stress test is the biggest cost center (8 parallel calls = ~40% of total)
- All phases use structured JSON output

TECH STACK:
- Python 3.13, Anthropic SDK, Pydantic models, DuckDuckGo search (free), HackerNews API (free)
- No database — JSON file storage for memory and runs
- CLI-only interface (argparse)

KNOWN LIMITATIONS (areas to improve):
- Pipeline is sequential except stress test — no parallelism in diverge/converge
- No way to resume a failed run — must restart from scratch
- No interactive mode — user reviews report after completion, no mid-pipeline feedback
- Memory grows unbounded — no pruning or compression
- No A/B testing of prompt variations
- No way to compare results across runs side-by-side
- No confidence estimation for individual scoring dimensions
- Single LLM provider (Anthropic only)
- Specify pipeline is a single LLM call — no iterative refinement

GOAL: Think of concrete, implementable improvements to xBrain that would make it more autonomous, produce higher-quality ideas, or improve the user experience. Prioritize ideas that have the best ratio of quality improvement to implementation cost — we cannot explode the API spend per run. Improvements that make the tool smarter without adding API calls are especially valuable. Think beyond the obvious.
